พิมพ์คำถามหรืองานที่ต้องการ เช่น "เขียนอีเมลสมัครงาน", "แปลภาษา", "สรุปข้อความนี้"

⚙️ การตั้งค่าขั้นกลาง / Intermediate Settings

1
สร้าง Projects / Create Projects
ใช้ Projects เพื่อจัดกลุ่มการสนทนาที่เกี่ยวข้องกัน และ Claude จะจำบริบทตลอดการสนทนาในโปรเจกต์นั้น
2
ตั้ง System Prompt ใน Project
ใส่คำสั่งถาวรสำหรับโปรเจกต์ เช่น "คุณเป็น Senior DevOps Engineer ที่เชี่ยวชาญ AWS และ Kubernetes ตอบเป็นภาษาไทยเสมอ"
# ตัวอย่าง System Prompt สำหรับ Project (IT)
คุณเป็น Senior Software Engineer ที่เชี่ยวชาญ Python, FastAPI, PostgreSQL
- ตอบเป็นภาษาไทยเสมอ พร้อม code ตัวอย่างทุกครั้ง
- เน้น clean code, SOLID principles และ test coverage
- บอก trade-off และ best practice ทุกครั้งที่ suggest solution
3
อัปโหลด Knowledge Files
อัปโหลดเอกสารอ้างอิงไว้ใน Project เช่น API Docs, Architecture Diagrams, Runbooks, ERD — Claude จะใช้ข้อมูลนี้ตลอดทุกการสนทนา
4
เลือก Model ที่ต้องการ

🚀 เทคนิคขั้นสูง / Advanced Techniques

🎭
Persona Stacking
กำหนดตัวตน + ความเชี่ยวชาญ + สไตล์การตอบพร้อมกัน เพื่อผลลัพธ์ที่แม่นยำยิ่งขึ้น
"คุณคือ Senior Backend Engineer ที่เชี่ยวชาญ Python & PostgreSQL มีประสบการณ์ 8 ปี ตอบแบบ concise ระบุ trade-off ทุกครั้ง"
🔗
Chain of Thought Prompting
ขอให้ Claude คิดทีละขั้น เพื่อลดข้อผิดพลาดในงาน debug หรือ architecture ที่ซับซ้อน
"ก่อนตอบ ให้ analyze root cause ทีละขั้น แล้วค่อยเสนอ solution พร้อม pros/cons"
📋
Output Format Control
ระบุ format ที่ต้องการ เช่น JSON, Markdown, YAML, SQL เพื่อนำไปใช้ต่อได้ทันที
"ตอบในรูปแบบ JSON เท่านั้น โดยมี keys: function_name, params, return_type, example"
🎯
Few-Shot Examples
ยกตัวอย่าง input-output ที่ต้องการ 2-3 ชุดก่อนถามจริง ผลลัพธ์แม่นยำขึ้นมาก
Input: "list all users"
Output: SELECT * FROM users;

Input: "users joined this month"
Output: SELECT * FROM users WHERE ...

💻 Claude Code

Claude Code
Terminal CLI Tool · For Developers

เครื่องมือ Command Line ที่ทรงพลังสำหรับ Developer — ทำงานกับ code, files, และ terminal ได้โดยตรง

⚠️
ต้องการ: Node.js 18+, npm, และ Anthropic API Key ก่อนเริ่มใช้งาน
1
ติดตั้ง Claude Code / Install
# ติดตั้งผ่าน npm
npm install -g @anthropic-ai/claude-code

# ตรวจสอบการติดตั้ง
claude --version
2
ตั้งค่า API Key
# เพิ่มใน ~/.bashrc หรือ ~/.zshrc
export ANTHROPIC_API_KEY="sk-ant-..."

# หรือรันครั้งแรก Claude จะถามให้ใส่
claude
3
เริ่มใช้งาน / Start Using
# เข้าโปรเจกต์และเรียก Claude
cd my-project
claude

# หรือสั่งตรงๆ แบบ one-liner
claude "แก้ bug ใน app.py บรรทัด 42"

🤝 Claude Cowork

Claude Cowork
Desktop App · Team Automation

Desktop App ที่เชื่อมต่อ Claude กับ Tools ต่างๆ ในองค์กร — ทำงานอัตโนมัติ, จัดการไฟล์, สั่งงาน AI แบบ Agent

🚀 เริ่มต้นใช้งาน Claude Cowork / Getting Started

1
ดาวน์โหลดและติดตั้ง / Download & Install
ไปที่ claude.ai/download → เลือก Mac หรือ Windows → ติดตั้งและเปิดแอป → Login ด้วย Anthropic Account เดียวกับที่ใช้บน Web

✅ ต้องการ: macOS 12+ หรือ Windows 10+ และบัญชี Claude (Free/Pro)
2
เปิดใช้ Cowork Mode
เมื่อเปิดแอปแล้ว ให้มองหา icon 🤝 Cowork ใน sidebar ซ้ายมือ → คลิกเพื่อสลับโหมด

หากไม่เห็น: ไปที่ Settings ⚙️ → Features → เปิดใช้งาน Cowork (Research Preview)

⚠️ Cowork ยังอยู่ใน Research Preview — อาจมี update บ่อย
3
เลือก Project Folder / Select Folder
คลิกปุ่ม "Select Folder" → เลือก folder ที่ต้องการให้ Claude ทำงานด้วย

Claude จะสามารถ: อ่านไฟล์, เขียนไฟล์, สร้างไฟล์ใหม่ ใน folder นั้น

💡 แนะนำ: สร้าง folder เฉพาะสำหรับ Cowork เช่น Documents/Claude/Projects/ชื่องาน
✅ Claude จะจำ folder นี้ข้ามเซสชัน — ไม่ต้องเลือกใหม่ทุกครั้ง
4
สร้าง CLAUDE.md — ให้ Claude จำข้อมูลสำคัญ
CLAUDE.md คือ "บันทึกความจำ" ของ Claude — อ่านอัตโนมัติทุก session

วิธีสร้าง: สั่ง Claude ว่า "สร้างไฟล์ CLAUDE.md สำหรับโปรเจกต์นี้ บันทึกว่าฉันชื่อ [ชื่อ] ทำงานที่ [บริษัท] โปรเจกต์นี้คือ [อธิบาย]"

🧠 Claude จะจำทุกอย่างใน CLAUDE.md โดยไม่ต้องบอกซ้ำ
5
ลองสั่งงานครั้งแรก
พิมพ์คำสั่งเป็นภาษาธรรมชาติ — Claude จะทำให้อัตโนมัติ ไม่ต้องรู้โค้ด

💡 Tips & Techniques

เทคนิคการใช้งานที่ได้ผล
Proven Tips & Techniques

เทคนิคเหล่านี้ช่วยให้ได้ผลลัพธ์ที่ดีขึ้น ประหยัดเวลา และลดต้นทุน

🎯
1. ให้ Context ที่ชัดเจน
บอก stack, version, environment ให้ครบ — Claude จะตอบได้ตรงกว่าหลายเท่า
❌ "แก้ bug หน่อย"
✅ "ฉันใช้ FastAPI 0.110 + PostgreSQL 15 + Python 3.12
function นี้ throw KeyError เมื่อ user_id ไม่มีใน dict
ต้องการ fix ให้ handle gracefully พร้อม log error"
📐
2. ระบุ Format ผลลัพธ์
บอกว่าต้องการ code, YAML, JSON, diagram หรือ bullet — Claude จะ output ตรงตาม format ที่ใช้ต่อได้เลย
"ตอบในรูปแบบ:
1. สาเหตุของปัญหา (1–3 บรรทัด)
2. Code fix พร้อม comment
3. Unit test 2 กรณีสำหรับ function นั้น"
🔄
3. ใช้ Iteration แก้ไขต่อเนื่อง
อย่าพอใจผลแรก — refine ต่อเนื่องใน conversation เดิม Claude จำ context ทั้งหมดไว้
"ดี แต่ขอเพิ่ม error handling ด้วย"
"Refactor ให้เป็น async/await แทน"
"เพิ่ม type hints และ docstring ด้วย"
🧩
4. แบ่งงานใหญ่เป็นขั้น
งาน system design ควรแบ่งเป็นขั้น ทีละงาน ผลลัพธ์ดีกว่ามาก
ขั้น 1: "ออกแบบ database schema สำหรับ e-commerce"
ขั้น 2: "ออกแบบ REST API endpoints จาก schema นั้น"
ขั้น 3: "เขียน FastAPI code สำหรับ endpoint /orders"
💾
5. ใช้ Projects จัดกลุ่ม codebase
สร้าง Project แยกสำหรับแต่ละ repo — ใส่ System Prompt และ architecture docs ไว้ Claude จะมี context ทุกครั้ง
Project: "Backend API v2"
System: "คุณเป็น Senior Dev ที่รู้จัก codebase นี้ดี..."
Files: architecture.md, api_spec.yaml, erd.png
🌐
6. แปะ Error Message เต็มๆ
Copy stack trace ทั้งหมด อย่า paraphrase — Claude debug ได้แม่นกว่ามากเมื่อเห็น error จริงๆ
✅ วาง stack trace เต็มๆ รวมถึง:
- Error type และ message
- File path และ line number
- Request payload (ถ้ามี)
📊
7. ขอ Code Review แบบ Structured
แทนที่จะถามว่า "code นี้ดีไหม" — ขอให้ review แบบมี category ได้ผลที่ actionable กว่ามาก
"Review code นี้ในด้าน:
1. Security vulnerabilities
2. Performance bottlenecks
3. SOLID principles
4. Missing edge cases
ระดับ: Production-ready หรือยัง?"
🎭
8. กำหนด Expertise Level
บอก Claude ว่าคุณมี level ไหน — Senior Dev อธิบายสั้น, Junior Dev อธิบายละเอียด
"อธิบายสำหรับ Junior Dev ที่เพิ่งเริ่มใช้ Docker"
"สมมติว่าฉัน Senior หยุดอธิบาย basic ต้องการเฉพาะ edge cases"
"อธิบายแบบ code ล้วน ไม่ต้องมี prose"
📎
9. แนบ Code / Config ทั้งหมด
อัปโหลดไฟล์ หรือ paste code เต็มๆ — Claude วิเคราะห์ได้แม่นกว่าการอธิบายด้วยคำพูด
แนบ: docker-compose.yml + Dockerfile + nginx.conf
แล้วถาม: "ทำไม container ไม่ communicate กัน?
และ optimize performance ของ nginx config ด้วย"
⚡
10. Prompt Template สำหรับงาน Repeat
งานที่ทำซ้ำ เช่น PR description, commit message, test case — เขียน template ครั้งเดียว ใช้ได้เรื่อยๆ
Template: "เขียน PR description สำหรับ:
- Changes: [อธิบาย]
- Breaking change: [Yes/No + detail]
- Test coverage: [%]
- Reviewer should focus on: [area]"
🔍
11. ขอให้ Claude อธิบาย code ก่อนแก้
ก่อน fix bug ขอให้ Claude อธิบาย code flow ก่อน — จะเข้าใจ root cause และแก้ได้ถูกจุดกว่า
"อธิบาย code flow นี้ก่อนว่าทำอะไร
แล้วค่อยบอกว่า bug อยู่ที่ไหน
แล้วค่อย propose fix พร้อม explain rationale"
🚀
12. Pro: ขอ Alternative Approaches
อย่าพอใจ solution แรก — ขอ 3 approaches พร้อม trade-off เพื่อเลือก approach ที่เหมาะกับ constraint จริงๆ
"ให้ 3 approach สำหรับ caching strategy นี้:
แต่ละ approach: ชื่อ, Pros, Cons,
เหมาะกับ use case ไหน, ความยากใน implement
Stack: Redis + FastAPI + PostgreSQL"
✍️ Prompt Engineering

Prompt Engineering
เขียน Prompt อย่างไรให้ได้ผลดีที่สุด

✅ DO — ควรทำ

ระบุบทบาท Role
"คุณเป็น Senior DevOps Engineer ที่เชี่ยวชาญ Kubernetes, Terraform และ AWS — ช่วย review deployment config นี้และบอก security risks ที่พบ..."
บอก Context ครบ
"โปรเจกต์นี้เป็น microservices ใช้ FastAPI, PostgreSQL, Redis, deploy บน AWS EKS — ฉันต้องการ optimize query นี้ที่ใช้เวลา 3 วินาที เป้าหมายคือ < 200ms"
ระบุข้อจำกัด Constraints
"เขียน Python เท่านั้น ไม่ใช้ library นอกจาก standard library, ต้องผ่าน type checking ด้วย mypy, มี docstring ทุก function, test coverage > 90%"
ให้ตัวอย่าง Examples
"เขียน unit test ในสไตล์นี้: [วาง test ที่มีอยู่เป็นตัวอย่าง] แล้วเขียน test เพิ่มให้ครอบคลุม edge cases ของ function parse_user_input() นี้..."

❌ DON'T — ไม่ควรทำ

Prompt คลุมเครือ
"แก้ code นี้หน่อย" — ไม่รู้ว่า bug อะไร, ภาษาอะไร, expected behavior คืออะไร, environment เป็นยังไง
ถามหลายอย่างพร้อมกัน
"ช่วย design DB + เขียน API + เขียน test + ทำ CI/CD + deploy ทีเดียว" — ควรแบ่งเป็นขั้นๆ ทีละ task
ใช้คำสั่งเชิงลบอย่างเดียว
"อย่าใช้ loop" — ควรบอกด้วยว่า "ใช้ list comprehension หรือ built-in functions เช่น map(), filter() แทน"
ไม่บอก Output Format
"ทำ code review" — ควรบอกว่าต้องการ inline comments, summary report, หรือ PR description ในรูปแบบ markdown
💡
สูตร Prompt ที่ได้ผลสำหรับงาน IT: [Role: Senior Dev/DevOps/Architect] + [Stack & Version] + [งานที่ต้องการ + expected behavior] + [Constraints: language, framework, style, test coverage] + [Output format: code/YAML/JSON/diagram/PR desc] = ผลลัพธ์ที่ดีขึ้น 10x

Key Terms Explained in Plain Thai

AI Model / โมเดล AI
"สมอง" ของ Claude — Haiku, Sonnet, Opus คือโมเดลต่างกัน เหมือนรถหลายรุ่น ฉลาดต่างกัน ราคาต่างกัน เลือกตามงาน
Prompt Prompt / คำสั่ง
ข้อความที่คุณพิมพ์ถาม Claude — ยิ่ง prompt ชัดเจน ยิ่งได้คำตอบที่ดี เหมือนการสั่งงานพนักงานใหม่ที่ต้องบอกรายละเอียด

การเชื่อมต่อระหว่าง Claude กับ app เฉพาะ เช่น "Google Drive Connector" — ทำให้ Claude อ่าน เขียน และจัดการไฟล์ใน Drive ได้โดยตรง

❓ FAQ

คำถามที่พบบ่อย & แก้ปัญหา
Troubleshooting Guide

ปัญหาที่ทีมพบบ่อยและวิธีแก้ไขอย่างรวดเร็ว

😕 Claude ตอบเป็นภาษาอังกฤษแทนที่จะเป็นไทย ▼
สาเหตุ: Claude เดาภาษาจากคำถาม — ถ้า Prompt มีคำอังกฤษมาก Claude มักตอบอังกฤษ

วิธีแก้ (3 วิธี):
1️⃣ เพิ่มท้าย Prompt: "ตอบเป็นภาษาไทยทั้งหมด"
2️⃣ ตั้ง System Prompt: "ตอบเป็นภาษาไทยเสมอ ยกเว้นชื่อ brand และ technical terms"
3️⃣ พิมพ์ต่อ: "ขอคำตอบเดิมเป็นภาษาไทย"

✅ วิธีที่ดีที่สุดคือใส่ใน System Prompt ของ Project — ทำครั้งเดียว ใช้ได้ตลอด
⚠️ Claude บอกว่า "Usage Limit Reached" หรือ quota หมด ▼
สาเหตุ: แต่ละ plan มี message limit รายวัน/รายเดือน — Haiku ใช้ quota น้อยกว่า Opus มาก

วิธีแก้:
1️⃣ สลับไปใช้ Model ที่ใช้ quota น้อยกว่า (Haiku สำหรับงานง่าย)
2️⃣ รอ quota reset (รีเซ็ตทุกวัน เที่ยงคืน UTC)
3️⃣ Upgrade plan หรือติดต่อ admin เพื่อขอ quota เพิ่ม

💡 ประหยัด quota: งานสั้นๆ เช่น แปลภาษา ตรวจสะกด → ใช้ Haiku; งานวิเคราะห์ → Sonnet
🔌 Connector เชื่อมต่อแล้วแต่ Claude ยังเข้าถึงไฟล์ไม่ได้ ▼
สาเหตุที่พบบ่อย:
- ยังไม่ได้ grant permission ให้ครบ (ต้องกด "Allow All" ไม่ใช่แค่ "Allow")
- Token หมดอายุ (OAuth token expire ทุก 60–90 วัน)
- ไฟล์อยู่ใน Shared Drive ที่ permission ต่างกัน

วิธีแก้:
1️⃣ ไปที่ Settings → Connectors → Disconnect → เชื่อมใหม่อีกครั้ง
2️⃣ ตรวจสอบว่า Google Account ที่ใช้ login ตรงกับที่มีไฟล์
3️⃣ ลอง "Refresh Connection" ใน Connector settings

ℹ️ Connector ส่วนใหญ่ต้อง login ใหม่ทุก 60–90 วัน เป็นเรื่องปกติ
📄 Claude อ่านไฟล์ไม่ได้ หรืออ่านแล้วผลลัพธ์ไม่ถูกต้อง ▼
สาเหตุ:
- ไฟล์ PDF ที่ scan มา (image-based) Claude อ่านตัวอักษรไม่ออก
- Excel ที่มี formula ซับซ้อนหรือ merged cells
- ไฟล์ขนาดใหญ่เกิน limit

วิธีแก้:
1️⃣ PDF scan → แปลงเป็น searchable PDF ก่อน (ใช้ Adobe Acrobat หรือ Google Drive)
2️⃣ Excel → Export เป็น CSV แล้วอัปโหลดใหม่
3️⃣ ไฟล์ใหญ่ → แบ่งเป็นส่วนๆ แล้วส่งทีละส่วน

✅ Format ที่ Claude อ่านได้ดีที่สุด: .txt, .md, .csv, searchable PDF, .docx
🤖 Claude ตอบผิด หรือข้อมูลไม่ตรงกับความเป็นจริง ▼
Claude อาจ "hallucinate": คิดคำตอบขึ้นมาเมื่อไม่รู้จริง — โดยเฉพาะตัวเลข, วันที่, ราคา, ชื่อคน

วิธีป้องกัน:
1️⃣ ให้ข้อมูลจริงแนบมาใน Prompt (เช่น "ราคาจาก price list นี้: ...")
2️⃣ ขอให้ Claude ระบุความมั่นใจ: "ถ้าไม่แน่ใจให้บอกว่าไม่รู้"
3️⃣ ข้อมูลสำคัญ → ตรวจสอบกับแหล่งข้อมูลจริงเสมอ

⚠️ Claude ไม่ใช่ Search Engine — ไม่รู้ข้อมูลหลัง knowledge cutoff และไม่รู้ข้อมูลภายในองค์กรของคุณ ถ้าไม่ได้แนบให้
⏰ Scheduled Task ไม่รันตามเวลาที่ตั้งไว้ ▼
สาเหตุที่พบบ่อย:
- Claude Cowork app ไม่ได้เปิดหรือ background process ถูก kill
- Timezone ไม่ตรง (ต้องตั้งเป็น Asia/Bangkok)
- Connector ที่ Task ใช้ expired token

วิธีแก้:
1️⃣ เปิด Cowork ทิ้งไว้ (หรือตั้ง auto-start)
2️⃣ ตรวจ Timezone ใน Settings → General
3️⃣ ตรวจ Connector ที่เชื่อมกับ Task ว่ายังใช้งานได้
4️⃣ ลอง "Run Now" เพื่อทดสอบว่า Task ทำงานได้ก่อนตั้งเวลา
🔒 ข้อมูลของเราปลอดภัยไหม? Claude เห็นข้อมูลใน Drive เราไหม? ▼
หลักการของ Anthropic:
- Claude เข้าถึงไฟล์ได้เฉพาะเมื่อคุณสั่งให้เข้าถึงในการสนทนานั้นๆ
- ไม่ใช้ข้อมูลของ Claude Pro/Team/Enterprise เพื่อ train model
- Connector permissions สามารถ revoke ได้ทุกเมื่อใน Settings

Best Practices สำหรับทีม:
1️⃣ ไม่แชร์ข้อมูลลับในการสนทนาที่ไม่จำเป็น
2️⃣ ใช้ Team/Enterprise plan สำหรับข้อมูลภายในองค์กร
3️⃣ Review Connector permissions ทุก 3 เดือน
