💬 Claude Chat

Claude Chat
Web & Mobile App · claude.ai

เข้าถึงได้ทันทีผ่าน claude.ai — ไม่ต้องติดตั้ง ไม่ต้องมีความรู้ด้านเทคนิค เหมาะกับทุกคน

1
เข้าเว็บไซต์ / Go to Website
เปิด claude.ai ในเบราว์เซอร์ สมัครด้วย Google หรืออีเมล ใช้ฟรีได้เลย
2
เริ่มสนทนา / Start Chatting
พิมพ์คำถามหรืองานที่ต้องการ เช่น "เขียนอีเมลสมัครงาน", "แปลภาษา", "สรุปข้อความนี้"
3
แนบไฟล์หรือรูปภาพ / Attach Files
กดปุ่ม 📎 เพื่อแนบ PDF, Word, รูปภาพ Claude สามารถวิเคราะห์เนื้อหาให้ได้
✅
Plan Free ทำอะไรได้? — ใช้ Claude Sonnet ฟรี, จำนวนข้อความจำกัด, อัปโหลดไฟล์ได้, สร้าง Project ได้
💬 สนทนาภาษาไทย-อังกฤษ
📄 วิเคราะห์เอกสาร PDF/Word
🖼️ วิเคราะห์รูปภาพ

🔢 คำนวณและวิเคราะห์ข้อมูล

⚙️ การตั้งค่าขั้นกลาง / Intermediate Settings

1
สร้าง Projects / Create Projects
ใช้ Projects เพื่อจัดกลุ่มการสนทนาที่เกี่ยวข้องกัน และ Claude จะจำบริบทตลอดการสนทนาในโปรเจกต์นั้น
2
ตั้ง System Prompt ใน Project
ใส่คำสั่งถาวรสำหรับโปรเจกต์ เช่น "คุณเป็น Senior DevOps Engineer ที่เชี่ยวชาญ AWS และ Kubernetes ตอบเป็นภาษาไทยเสมอ"
# ตัวอย่าง System Prompt สำหรับ Project (IT)
คุณเป็น Senior Software Engineer ที่เชี่ยวชาญ Python, FastAPI, PostgreSQL
- ตอบเป็นภาษาไทยเสมอ พร้อม code ตัวอย่าง
- เน้น clean code, SOLID principles
- บอก trade-off และ best practice ทุกครั้ง
3
อัปโหลด Knowledge Files
อัปโหลดเอกสารอ้างอิงไว้ใน Project เช่น API Docs, Architecture Diagrams, Runbooks, ERD — Claude จะใช้ข้อมูลนี้ตลอดทุกการสนทนา
4
เลือก Model ที่ต้องการ
กด Model Selector ที่ด้านบนของ Chat เลือก Haiku (เร็ว), Sonnet (สมดุล), หรือ Opus (ฉลาดสุด)

🚀 เทคนิคขั้นสูง / Advanced Techniques

🎭
Persona Stacking
กำหนดตัวตน + ความเชี่ยวชาญ + สไตล์การตอบพร้อมกัน เพื่อผลลัพธ์ที่แม่นยำยิ่งขึ้น
"คุณคือ Senior Backend Engineer ที่เชี่ยวชาญ Python & PostgreSQL มีประสบการณ์ 8 ปี ตอบแบบ concise ระบุ trade-off ทุกครั้ง"
🔗
Chain of Thought Prompting
ขอให้ Claude คิดทีละขั้น เพื่อลดข้อผิดพลาดในงาน debug หรือ architecture ที่ซับซ้อน
"ก่อนตอบ ให้ analyze root cause ทีละขั้น แล้วค่อยเสนอ solution พร้อม pros/cons"
📋
Output Format Control
ระบุ format ที่ต้องการ เช่น JSON, Markdown, YAML, SQL เพื่อนำไปใช้ต่อได้ทันที
"ตอบในรูปแบบ JSON เท่านั้น โดยมี keys: function_name, params, return_type, example"
🎯
Few-Shot Examples
ยกตัวอย่าง input-output ที่ต้องการ 2-3 ชุดก่อนถามจริง ผลลัพธ์แม่นยำขึ้นมาก
Input: "list all users"
Output: SELECT * FROM users;

Input: "users joined this month"
Output: SELECT * FROM users WHERE ...

# ถามต่อ: Input: "active admin users" → ?
💻 Claude Code

Claude Code
Terminal CLI Tool · For Developers

เครื่องมือ Command Line ที่ทรงพลังสำหรับ Developer — ทำงานกับ code, files, และ terminal ได้โดยตรง

⚠️
ต้องการ: Node.js 18+, npm, และ Anthropic API Key ก่อนเริ่มใช้งาน
1
ติดตั้ง Claude Code / Install
# ติดตั้งผ่าน npm
npm install -g @anthropic-ai/claude-code

# ตรวจสอบการติดตั้ง
claude --version
2
ตั้งค่า API Key
# เพิ่มใน ~/.bashrc หรือ ~/.zshrc
export ANTHROPIC_API_KEY="sk-ant-..."

# หรือรันครั้งแรก Claude จะถามให้ใส่
claude
3
เริ่มใช้งาน / Start Using
# เข้าโปรเจกต์และเรียก Claude
cd my-project
claude

# หรือสั่งตรงๆ แบบ one-liner
claude "แก้ bug ใน app.py บรรทัด 42"

⚙️ CLAUDE.md — Memory File

สร้างไฟล์ CLAUDE.md ในโปรเจกต์ — Claude จะอ่านอัตโนมัติทุกครั้งที่เริ่มการสนทนา
# CLAUDE.md ตัวอย่าง
## Project Overview
- ชื่อโปรเจกต์: BFF E-commerce Platform
- Stack: Next.js, Supabase, Tailwind
- ภาษา: TypeScript

## Coding Rules
- ใช้ async/await เสมอ (ห้ามใช้ .then())
- ตั้งชื่อตัวแปรเป็น camelCase
- เขียน comment เป็นภาษาไทย

## Commands
- Run dev: npm run dev
- Test: npm test
- Build: npm run build

# วิธีตั้งค่า API Key อย่างปลอดภัย

## macOS/Linux — เพิ่มใน ~/.zshrc
export ANTHROPIC_API_KEY="sk-ant-api03-..."

## Windows — PowerShell
$env:ANTHROPIC_API_KEY = "sk-ant-api03-..."

## หรือใช้ .env file (อย่า commit ขึ้น Git!)
ANTHROPIC_API_KEY=sk-ant-api03-...
⚠️

1. ให้ Context ที่ชัดเจน
บอก Background ว่าคุณเป็นใคร ทำอะไร เพื่ออะไร — ยิ่งชัด ยิ่งตอบตรง
❌ "แก้ bug หน่อย"
✅ "ฉันเป็น Backend Dev ใช้ FastAPI + PostgreSQL
function นี้ throw KeyError เมื่อ user_id ไม่มีใน dict
ต้องการ fix ให้ handle gracefully พร้อม log error"
📐
2. ระบุ Format ผลลัพธ์
บอกว่าต้องการผลลัพธ์ในรูปแบบอะไร เพื่อนำไปใช้ต่อได้ทันที
"ตอบในรูปแบบตาราง Markdown
3 ตัวเลือก architecture แต่ละตัวมี:
ชื่อ, Pros, Cons, เมื่อไหรควรใช้"
🔄
3. ใช้ Iteration
อย่ากลัวที่จะ refine — พิมพ์ follow-up เพื่อ improve code หรือ solution ได้ต่อเนื่อง
"ดี แต่ขอเพิ่ม error handling ด้วย"
"เพิ่ม unit test ให้ครอบคลุม edge cases"
"refactor ให้เป็น async/await แทน"
🧩
4. แบ่งงานใหญ่เป็นชิ้นเล็ก
งาน system design ควรแบ่งเป็นขั้นตอน แต่ละขั้นสั่งแยกกัน ผลลัพธ์ดีกว่ามาก
ขั้น 1: "ออกแบบ database schema สำหรับ e-commerce"
ขั้น 2: "ออกแบบ API endpoints จาก schema นั้น"
ขั้น 3: "เขียน FastAPI code สำหรับ endpoint /orders"
💾
5. ใช้ Projects ให้เป็นประโยชน์
สร้าง Project แยกสำหรับแต่ละ repo ใส่ System Prompt และ docs อ้างอิงไว้ — Claude จะมี context ทุกครั้ง
Project: "Backend API v2"
System: "คุณเป็น Senior Dev ที่รู้จัก codebase นี้ดี..."
Files: architecture.md, api_spec.yaml, erd.png
💰
6. ประหยัด Token / Cost
ใช้ Haiku สำหรับงาน code review ง่ายๆ ประหยัดได้มากเมื่อใช้ volume สูงใน CI/CD pipeline
# เปลี่ยน Model ตามงาน
Code linting → Haiku (เร็ว ถูก)
Code review → Sonnet (สมดุล)
Architecture design → Opus (ฉลาดสุด)
✍️ Prompt Engineering

Prompt Engineering
เขียน Prompt อย่างไรให้ได้ผลดีที่สุด

✅ DO — ควรทำ

ระบุบทบาท Role
"คุณเป็น Senior DevOps Engineer ที่เชี่ยวชาญ Kubernetes, Terraform และ AWS — ช่วย review deployment config นี้..."
บอก Context ครบ
"โปรเจกต์นี้เป็น microservices ใช้ FastAPI, PostgreSQL, Redis, deploy บน AWS EKS — ฉันต้องการ optimize query นี้ที่ใช้เวลา 3 วินาที..."
ระบุข้อจำกัด Constraints
"เขียน Python เท่านั้น ไม่ใช้ library นอกจาก standard library, ต้องผ่าน type checking ด้วย mypy, มี docstring ทุก function"
ให้ตัวอย่าง Examples
"เขียน unit test ในสไตล์นี้: [ตัวอย่าง test ที่มีอยู่] แล้วเขียน test เพิ่มให้ครอบคลุม edge cases ของ function นี้..."

❌ DON'T — ไม่ควรทำ

Prompt คลุมเครือ
"แก้ code นี้หน่อย" — ไม่รู้ว่า bug อะไร, ภาษาอะไร, expected behavior คืออะไร
ถามหลายอย่างพร้อมกัน
"ช่วย design DB + เขียน API + เขียน test + deploy script ทีเดียว" — ควรแบ่งเป็นขั้นๆ ทีละงาน
ใช้คำสั่งเชิงลบอย่างเดียว
"อย่าใช้ loop" — ควรบอกด้วยว่า "ใช้ list comprehension หรือ built-in functions แทน"
ไม่บอก Output Format
"ทำ code review" — ควรบอกว่าต้องการ inline comments, summary report, หรือ PR description
💡
สูตร Prompt ที่ได้ผลสำหรับงาน IT: [Role: Senior Dev/DevOps/Architect] + [Stack & Version] + [งานที่ต้องการ + expected behavior] + [Constraints: language, framework, style] + [Output format: code/YAML/JSON/diagram] = ผลลัพธ์ที่ดีขึ้น 10x
🎯 Skills

Skills — ทักษะพิเศษของ Claude
How to Use, Add & Invoke Skills

Skills คือ "โปรแกรมสำเร็จรูป" ที่สอน Claude ให้ทำงานเฉพาะทาง เช่น สร้าง Excel, สร้าง Word, วิเคราะห์ข้อมูล, ทำ Presentation — เรียกใช้ได้ทันทีด้วยคำสั่งเดียว

📋
Skill คืออะไร?
ไฟล์ SKILL.md ที่เก็บ instructions ให้ Claude ทำงานเฉพาะทาง — เหมือน "สูตรอาหาร" ที่ Claude ทำตามได้ทันที
⚡
ทำไมต้องใช้ Skill?
ประหยัดเวลา ไม่ต้องพิมพ์ prompt ซ้ำๆ งานซับซ้อนทำได้ในคำสั่งเดียว ทีมใช้ร่วมกันได้
🏪
Skills มาจากไหน?
Built-in Skills (มาพร้อม Cowork), Plugin Skills (ติดตั้งเพิ่ม), Custom Skills (สร้างเอง)

🔍 Connector vs MCP — ต่างกันอย่างไร?
🤝 Connector (Cowork)
✅ ติดตั้งง่าย ผ่าน GUI
✅ ไม่ต้องเขียน code
✅ OAuth แบบ one-click
✅ เหมาะกับ non-developer
⚡ ใช้ได้ใน Cowork mode เท่านั้น
🔧 MCP Server (Code)
✅ Flexible กว่า ปรับแต่งได้
✅ สร้าง custom MCP เองได้
✅ ทำงานร่วมกับ Claude Code
✅ Control ระดับ permission ละเอียด
⚡ ต้องการความรู้ technical

สร้าง Projects ใหม่และวาง System Prompts
เปิด projects.json จาก export → Copy "system_prompt" ของแต่ละ Project → สร้าง Project ใหม่ใน account ใหม่ → วาง System Prompt
4
Upload Knowledge Files กลับ
Upload ไฟล์อ้างอิงต่างๆ ที่เคยมีใน Projects เดิม กลับเข้า Projects ใหม่
5
Reconnect Plugins & Connectors
ติดตั้ง Plugins อีกครั้งใน account ใหม่ → ทำ OAuth ใหม่ → ตรวจสอบว่าทุก Connector ทำงานได้
🔄 Chat ↔ Code (ใช้ร่วมกันแบบ Hybrid Workflow) ▼
Workflow ที่ใช้ทั้ง Chat และ Code ร่วมกันในโปรเจกต์เดียว โดยไม่ต้อง migrate จริงๆ
💬 Claude Chat → brainstorm, design, ตัดสินใจ architecture
⬇️ sync via CLAUDE.md → Copy decisions ลง CLAUDE.md
💻 Claude Code → implement, test, debug, deploy
⬆️ sync via Chat → report ผลลัพธ์กลับมาใน Chat Project
🔄 Repeat cycle → iterate ต่อเนื่อง
💡
Best Practice: ใช้ CLAUDE.md เป็น "Single Source of Truth" — อัปเดตทุกครั้งที่มีการตัดสินใจสำคัญใน Chat แล้ว Claude Code จะ pick up อัตโนมัติ
